# v0.6.0-rc.16

RC.15 reached the real endpoint-local public Service probes and exposed two
acceptance-verifier representation differences. The CCM, Cilium datapath,
public floating IP, TCP/80 path, UDP/443 path, and retained client address were
already correct.

BusyBox HTTP CGI reports an IPv4 client through its dual-stack listener as a
bracketed IPv4-mapped IPv6 address, for example `[::ffff:203.0.113.10]`.
The verifier now removes only the optional address brackets and converts only
IPv4-mapped IPv6 values to their canonical IPv4 address. Its existing policy
still requires one global IPv4 address, different from every edge floating IP
and identical across all probes. Native IPv6 and malformed values remain
rejected.

Kubernetes can serialize an EndpointSlice with no endpoints as
`"endpoints": null`. The withdrawal proof now treats a missing or explicit
null field as an empty endpoint set while continuing to reject every
non-array, non-null representation.

Focused regression tests run both in the local static gate and while building
the E2E runner image. No controller, CRD, chart runtime, or InSpace mutation
behavior changes in this candidate.

The RC.15 run used an in-place diagnostic verifier and therefore does not
qualify for stable promotion. Stable v0.6.0 remains gated on this exact,
unchanged RC.16 commit completing the full guarded real-cloud lifecycle and
restoring the isolated account to its immutable baseline.
