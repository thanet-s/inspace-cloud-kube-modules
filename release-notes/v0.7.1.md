## Dependency maintenance

v0.7.1 updates the CSI driver's gRPC dependency from 1.82.0 to 1.82.1,
Ansible Core in the release E2E environment from 2.21.1 to 2.21.2, and the
GitHub Actions release toolchain to `actions/setup-go` 7.0.0 and
`actions/attest` 4.2.0.

There are no controller API, CRD, Helm value, or controller runtime behavior
changes in this patch release.

The operator-facing `deploy/run.sh` lifecycle now runs from a
dependency-locked Docker image. Inventory remains a read-only runtime bind
mount, SSH material is mounted read-only, and durable lifecycle state is
mounted read-write without entering the image. Editing operator configuration
therefore does not rebuild the runner. The image is rebuilt only when its
versioned runner sources or dependency lock change. The runner builds natively
on both x86-64 and Apple Silicon; only the released InSpace bootstrap
controller remains explicitly pinned to `linux/amd64`.
