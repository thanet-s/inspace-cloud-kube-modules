# v0.6.0-rc.7

This release candidate fixes two live API compatibility findings from RC.6:

- bootstrap discovery now loads the TLS parser before validating the generated
  cache CA;
- floating-IP list and exact-read responses accept InSpace's live
  post-unassignment representation, where `assigned_to` is omitted and
  `unassigned_at` is present;
- VM deletion accepts the live API's JSON-bearing HTTP 200 as well as its empty
  success response. The body must be one complete, duplicate-free JSON value,
  and controllers still require authoritative VM absence before completing.

The floating-IP exception is endpoint-local and fail closed. Mutation responses
still require explicit assignment state, and an omitted relationship is
rejected unless the readback also contains an unassignment marker and no
contradictory resource relationship.
