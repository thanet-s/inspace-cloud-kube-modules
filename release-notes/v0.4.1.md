## Ubuntu package and DNS bootstrap

- Configures TOT as the primary Ubuntu mirror and KKU as the request-failure
  fallback for both regular and security suites on the bastion, control-plane
  nodes, and Karpenter workers.
- Replaces DHCP-provided DNS with static Google resolvers and stops and masks
  `systemd-resolved` before package and Kubernetes bootstrap.
- Advances fixed-node ownership to schema v4 and Karpenter bootstrap to schema
  v9 so the immutable configuration change is explicit and auditable.

## Breaking upgrade note

Existing v0.4.0 control-plane and bastion VMs use ownership schema v3 and are
not adopted in place. Destroy and recreate the cluster to apply v0.4.1. Safe
teardown of older v3-owned fixed VMs remains supported.

## Validation

- Unit contracts cover the exact APT sources, mirror priorities, resolver
  file, service masking, and ownership-schema behavior.
- The destructive release suite validates the configuration on real InSpace
  bastion, control-plane, and AMD EPYC Karpenter worker VMs and removes the
  paid public NLB immediately after its assertion.
