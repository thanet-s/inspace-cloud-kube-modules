## Ubuntu package, DNS, and hostname bootstrap

- Configures TOT as the primary Ubuntu mirror and KKU as the request-failure
  fallback for both regular and security suites on the bastion, control-plane
  nodes, and Karpenter workers.
- Replaces DHCP-provided DNS with static Google resolvers and stops and masks
  `systemd-resolved` before package and Kubernetes bootstrap.
- Replaces stale `127.0.1.1` host entries with each generated node hostname and
  verifies local resolution with bounded readback retry before package or
  resolver setup.

## Bootstrap cache reliability

- Derives the seed from the cluster's disabled RKE2 addons. The full audited
  inventory remains 34 images; disabling `rke2-ingress-nginx` removes its two
  unused images, producing the 32-image E2E seed.
- Retries both the writable seed-registry startup and final forced recreation
  of the read-only registry plus NGINX with bounded incremental backoff.

## Breaking upgrade note

Existing v0.4.0 control-plane and bastion VMs use ownership schema v3 and are
not adopted in place. The final v0.4.1 fixed-node schema is v6; RC.1 and RC.2
used the intermediate v4 and v5 schemas. Destroy and recreate the cluster to
apply v0.4.1. Safe teardown of supported older fixed-node records remains
available.

Karpenter bootstrap advances from v8 in v0.4.0 to v11 in the final v0.4.1
release; RC.1 and RC.2 used the intermediate v9 and v10 schemas. Existing
workers are eligible for drift replacement through the normal NodeClaim
lifecycle.

## Validation scope

- Unit and static contracts cover the APT sources, mirror priorities, resolver
  file, bounded hostname readback, cache inventory selection and retry bounds,
  and ownership/bootstrap schema behavior.
- The destructive release suite is configured to check these contracts on the
  bastion, control planes, and AMD EPYC Karpenter worker while keeping its paid
  public NLB short-lived.
