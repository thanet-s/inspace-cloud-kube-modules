# v0.6.0-rc.9

This release candidate fixes two acceptance-harness defects found by the RC.8
live run.

Every Ansible play now owns or explicitly receives each value it consumes.
Provisioning facts use phase-specific names, the bastion receives the immutable
release manifest through its dynamic inventory, and the controller-install
play independently reloads and verifies the persisted release manifest. The
Karpenter test resources render only from the persisted topology. This removes
the hidden same-host fact precedence that masked incomplete later-play
variables and fixes the missing SSH configuration path that stopped RC.8 after
the three-node control plane was ready.

Cleanup cloud audits now expose semantic `bootstrap-only` and `zero`
expectations through their process status. Ansible retries that status without
parsing stdout from a failed HTTP attempt, so a transient InSpace transport
failure cannot abort the retry loop before cleanup converges. The audit still
prints canonical JSON, requires three stable strict reads, validates exact
bootstrap roles, and fails closed on malformed results.

RC.8 live acceptance created the bastion and three fixed RKE2 servers, verified
the immutable cache and controller digests, reached RKE2 v1.35.6 readiness with
embedded etcd, and proved private kube-vip API failover without interruption.
Automatic recovery then removed all four VMs, four floating IPs, and both
firewalls. An independent three-read account inventory comparison reported
zero extra or missing resources across VMs, floating IPs, firewalls, load
balancers, disks, networks, buckets, and service packages.
