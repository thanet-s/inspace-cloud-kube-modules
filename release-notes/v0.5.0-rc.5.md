## NodeLB withdrawal convergence

RC.5 fixes two failures found during destructive RC.4 acceptance.

Service deletion now retains its consecutive authoritative firewall-absence
proof after the datapath activation marker is cleared. The same proof can
therefore advance into firewall deletion and finalizer removal instead of
cycling back to its first confirmation indefinitely.

Emergency firewall withdrawal now persists a per-firewall/per-VM detach ledger
on the parent Service before calling InSpace. Repeated reconciliations and CCM
restarts do not repeat an unassign while cloud readback remains stale. New
assignment pairs are detached immediately, while an unchanged pair is retried
only after a five-minute safety interval. Partial readback convergence cannot
re-arm pairs that were already handled.

Recovery clears a ledger entry only after a non-empty, exact assignment
readback for that same current-firewall/Ready-VM pair. Entries for old or
unrelated firewalls remain fenced, and a zero-node empty readback is not
accepted as recovery proof. Public status waits for a separate assignment
re-audit after the durable recovery update.

## Validation scope

Unit regressions cover stale readback, controller restart, partial assignment
expansion and shrink, delayed retry, cloud errors, zero Ready nodes, scoped
recovery, and deletion finalizer convergence. Destructive RC.5 live acceptance
is still required; these notes do not claim that it has passed.
