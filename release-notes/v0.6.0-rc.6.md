# v0.6.0-rc.6

This release candidate fixes the live E2E preflight audit introduced by the
crash-safe ownership journal hardening. A new run may now prove a stable,
exactly corroborated zero-resource baseline before `state.json` exists.

The exception remains fail closed: a missing journal is accepted only when the
three strict audit reads find zero owned resources. Any discovered resource, or
any post-mutation audit without the journal, is rejected.
