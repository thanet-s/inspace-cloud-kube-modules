## Stable promotion

The controller and chart sources in v0.4.0 are unchanged from
v0.4.0-rc.2. Changes after RC.2 only strengthen the release-acceptance
harness and document this promotion.

The published RC.2 artifacts passed the complete isolated-account lifecycle:
three fixed RKE2 control planes, the private API VIP, the bastion bootstrap
cache, an AMD EPYC Karpenter worker, CCM target reconciliation, CSI RWO
persistence, two same-port Cilium L2 VIPs and failover, the public TCP NLB,
and deterministic teardown to a zero-resource cloud audit.

During acceptance, a worker received a source FIP that reproducibly timed out
against Docker Hub. The test retained that FIP and VM, provisioned a distinct
replacement, proved direct Docker Hub and GHCR pulls on the replacement, and
only then removed the rejected NodeClaim, VM, and FIP. This also live-proved
the bounded worker-egress recovery ordering added to the test harness.

## Breaking upgrade note

This release adds required `spec.bootstrapCache` contracts to both
`InSpaceCluster` and `InSpaceNodeClass`. For an existing direct-download
cluster, first upgrade the CRD chart, immediately add
`bootstrapCache.directDownload: true` to every NodeClass and to the persisted
InSpaceCluster configuration, and only then upgrade or run the new
controllers. The direct control-plane cloud-init bytes remain frozen to the
pre-cache v0.3.1 contract.

Existing bastions cannot be converted in place to cached mode because their
cloud-init and private address are immutable ownership inputs. Fresh clusters
should use the cached mode exercised by the release E2E.

## Private bootstrap cache

- Turns the existing bastion into the default private cache without allocating
  another VIP. Nodes bind `cache.<cluster>.inspace.internal` to the bastion's
  allocator-assigned VPC address and use TLS on TCP/8443.
- Pre-seeds the pinned RKE2 archive and 34 audited system images into a bounded
  10 GB filesystem, then serves the registry read-only. Arbitrary workload
  registries are not mirrored or rewritten.
- Generates deterministic ECDSA P-256 CA and server certificates from an
  operator-owned 256-bit key and the real persisted initialization timestamp.
  Both certificates expire exactly 15 calendar years after that instant.
- Makes Karpenter verify the private cache, including hostname/SNI and its
  pinned CA, before purchasing a worker. `directDownload: true` remains the
  explicit escape hatch.
- Lets the Helm chart rewrite only its own GHCR and Kubernetes CSI images to
  the private cache. Custom repositories remain unchanged.
- Uses a fail-closed Distribution Registry v3 read-only configuration and
  replaces that complete section only while seeding, avoiding silently
  writable steady-state serving.

## Karpenter catalog

- Adds `is-general-1c-2g` with 1 vCPU and 2 GiB RAM and `is-memory-1c-4g`
  with 1 vCPU and 4 GiB RAM.
- Adds five `extra-memory` shapes at 8 GiB/vCPU for CPU sizes
  `1, 2, 4, 6, 8`, bringing the catalog to 31 variants while respecting the
  InSpace 64 GiB per-VM RAM limit.
- An unconstrained general- or memory-family NodePool may now choose its
  cheaper 1-vCPU shape. Use `inspace.cloud/instance-cpu Gte ["2"]` when a
  2-vCPU floor is required.
- Karpenter prices CPU and RAM consistently across the equal-priced
  `amd-epyc` and `intel-scalable` host classes; host-class selection remains a
  NodePool requirement.

## Cloud load balancer improvements

- Public InSpace NLBs exclude nodes carrying
  `node-role.kubernetes.io/control-plane` or the legacy
  `node-role.kubernetes.io/master` label under both Cluster and Local traffic
  policies.
- Node readiness, role labels, local endpoints, the external-load-balancer
  exclusion label, disruption taints, and node deletion all trigger target
  reconciliation.
- `externalTrafficPolicy: Local` selects only eligible nodes with a ready local
  endpoint. An omitted policy remains Kubernetes' default, `Cluster`; the CCM
  does not rewrite the Service.

## Release acceptance

- Proves cache size limits, read-only serving, seeded assets, TLS identity and
  validity, private node routing, and direct-download isolation for ordinary
  workloads.
- Proves Karpenter worker creation and bounded FIP replacement, CSI block-volume
  attachment and persistence across Pod replacement, and CCM target removal
  and restoration.
- Proves two private Services can use the same TCP port on distinct Cilium L2
  VIPs, including lease-holder and ARP failover.
- Deletes the paid public NLB immediately after its checks, destroys the full
  cluster, and requires an independent zero-resource cloud audit.
