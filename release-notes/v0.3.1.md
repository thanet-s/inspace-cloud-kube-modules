## Breaking change

The v0.3 API does not support `InSpaceNodeClass.spec.hostPoolSelector`.
Hardware-class selection belongs exclusively to Karpenter `NodePool`
requirements through `inspace.cloud/host-class`; there is no compatibility
shim for selector-based NodeClasses from v0.2.x or earlier.

A NodePool without an explicit host-class requirement allows both equal-priced
`amd-epyc` and `intel-scalable` offerings and therefore does not guarantee AMD
EPYC. Fresh installations are unaffected when their NodePools declare the
intended class.

## Cloud load balancer improvements

- Public InSpace NLBs never target nodes carrying
  `node-role.kubernetes.io/control-plane` or the legacy
  `node-role.kubernetes.io/master` label.
- The exclusion applies to both `externalTrafficPolicy: Cluster` and `Local`.
- An omitted `externalTrafficPolicy` remains Kubernetes' default, `Cluster`;
  the CCM does not rewrite the Service. Set `Local` explicitly when
  endpoint-local targeting is required.
- Node readiness, role-label, endpoint, exclusion-label, disruption-taint, and
  deletion changes trigger target reconciliation.
