## Public node load balancers

- Adds `loadBalancerClass: inspace.cloud/node` as a lower-cost public load
  balancer backed by dedicated, tainted Karpenter nodes and Cilium Node IPAM.
  Existing private Cilium L2 Services and paid public InSpace NLBs remain
  supported.
- Defaults to `public-node-shared`: compatible TCP/UDP ports reuse a shard and
  a conflict is placed on another shard automatically. `public-node-dedicated`
  always receives its own shard and accepts an exact CPU and memory shape.
- Defaults each shard to one AMD EPYC, linux/amd64, 1-vCPU, 2-GiB node with a
  30-GiB root disk. `nodes-per-shard` can publish multiple independent FIPs.
- Requires a selected Service, `allocateLoadBalancerNodePorts: false`, and
  `externalTrafficPolicy: Cluster`, keeping application workloads off the
  public load-balancer nodes.

## Firewall and identity safety

- Keeps the existing private base firewall on every node-load-balancer VM.
  One cluster-owned firewall containing only portless inbound ICMP from Any is
  reused by all authorized node-load-balancer VMs. Each Service owns a separate
  TCP/UDP-only firewall, with `loadBalancerSourceRanges` applied only to those
  Service ports.
- Advertises a node only after its exact Node -> NodeClaim -> NodePool ->
  generated NodeClass ownership chain, active public FIP, Ready state, private
  base firewall, shared ICMP firewall, and every required Service firewall are
  confirmed from authoritative readback.
- Uses NodeRestriction-reserved readiness and identity labels and fails closed
  on Node, FIP, NodePool, or firewall drift. Cleanup retains finalizers until
  firewall/FIP/capacity absence is confirmed.
- A cluster using this feature is a trusted-admin boundary. RBAC or admission
  policy must reserve raw `io.cilium/node` Services, `externalIPs`, and the
  public node-load-balancer labels, tolerations, and selectors for the CCM.

## Bootstrap and compatibility

- Enables Cilium Node IPAM in generated RKE2 Cilium configuration.
- Adds the `public-node-load-balancer` Karpenter firewall profile while keeping
  omitted profiles backward-compatible with `private-worker`.
- Defaults an omitted bastion management CIDR to Any and adds one inbound ICMP
  rule to the bastion firewall; an explicit `/32` restricts both SSH and ping.

## Breaking upgrade note

Enabling Cilium Node IPAM changes immutable control-plane cloud-init and moves
the fixed control-plane ownership schema from v6 to v7. Existing v0.4.1 fixed
VMs are not adopted in place; destroy and recreate the cluster to use v0.5.0.
Teardown continues to recognize supported older records, including a v6
bastion paired with v7 control planes during an interrupted recreation.

This release candidate adds controller permissions for the generated
NodeClass, static NodePools, NodeClaims, and Cilium shadow Services. Upgrade the
chart and CRD chart together before creating an `inspace.cloud/node` Service.

## Candidate validation

- Unit and race suites cover shared/dedicated planning, port conflict sharding,
  ambiguous firewall creation, shared ICMP reuse and final cleanup, protected
  node identity, FIP ownership, drift fail-closed behavior, and Karpenter
  firewall-assignment validation.
- The guarded release E2E is configured to prove mixed TCP 80, TCP 443, and
  UDP 443 on a shared shard, real public responses and ping, automatic shard
  expansion, partial-member cleanup, exact cloud ownership, and final teardown
  to a zero-resource account.
