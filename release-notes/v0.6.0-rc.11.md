# v0.6.0-rc.11

RC.10 reached the real public-NLB acceptance path and exposed two
floating-IP response differences that are not represented by the published
examples. InSpace committed the deterministic floating-IP create but returned
HTTP 200 instead of 201. The new, never-assigned address then appeared in both
the list and exact-address endpoints with its complete stable identity but
without `assigned_to`, `assigned_to_resource_type`,
`assigned_to_private_ip`, or `unassigned_at`.

The RC.10 mutation fence behaved safely: it retained the issued create receipt,
did not send a second POST, and left the Kubernetes Service cleanup finalizer
in place. The guarded teardown also failed closed before deleting Kubernetes
owners because the strict account reader rejected the same sparse row. The
paid Service was explicitly moved to terminating state so it cannot add
targets or recreate resources, and its exact Service UID, deterministic names,
NLB UUID, and floating-IP address were durably retained for receipt-preserving
recovery.

RC.11 accepts only HTTP 200 or 201 for `CreateFloatingIP`; it does not broaden
the endpoint to arbitrary 2xx responses. Empty, partial, malformed, or
identity-incompatible success bodies remain non-authoritative. A fully sparse
unassigned floating-IP read is accepted only after the list and exact-address
endpoints independently corroborate the same canonical address, UUID, name,
and billing account. Partial or contradictory assignment metadata still fails
closed; a coherent explicit assignment returned by the corroborating read is
preserved. Assignment and metadata-update mutation responses keep their stricter
relationship contracts.

The destructive E2E reader recognizes the same coherent sparse unassigned
shape while continuing to reject stray resource-type or private-address
fields. Regression coverage includes the live HTTP-status matrix, two-read
sparse identity corroboration, malformed and conflicting rows, and cleanup of
a deleting paid-NLB Service whose issued create receipt resolves to the exact
new floating IP. Stable promotion remains gated on a complete guarded RC.11
real-cloud run and exact account-baseline teardown.
