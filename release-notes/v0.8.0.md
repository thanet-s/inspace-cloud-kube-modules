## Dedicated Cilium egress gateways

v0.8.0 enables Cilium Egress Gateway in the immutable RKE2 bootstrap
configuration. Selected workloads can now leave the cluster through dedicated
Karpenter nodes while remaining scheduled on ordinary workers.

The new `egress-gateway-static.yaml` chart example creates a two-replica static
NodePool with a permanent `NoSchedule` taint and a NodeRestriction-protected
gateway label. Its `CiliumEgressGatewayPolicy` selects the example
`payments/payment-gateway` workload and deliberately omits both `egressIP` and
`interface`, allowing Cilium to use the default-route interface. Each gateway
node's InSpace floating IPv4 is the public address to allowlist with an
external provider.

The two nodes are standby capacity rather than active-active gateways for one
policy. Cilium selects the first matching node in lexical name order; both
floating addresses should be allowlisted. Static capacity fixes the node count,
not the lifetime of a floating IP, so any explicitly permitted replacement
requires the new address to be added to the external allowlist.

## Upgrade notes

Enabling Egress Gateway changes fixed control-plane cloud-init, so new
control-plane ownership records use schema v9. Teardown remains compatible
with v8 control planes and the unchanged v6 bastion. Existing clusters are not
mutated in place: they must use the documented guarded destroy/recreate
lifecycle to receive the new bootstrap configuration.

The operator deployment example now selects module version `0.8.0`. The live
mutation-safety harness also isolates its fake API tests from a developer's
ignored `.env`; normal live commands continue loading `.env` unless their test
environment explicitly opts out.
