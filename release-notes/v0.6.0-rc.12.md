# v0.6.0-rc.12

RC.11 recovered the retained paid-NLB receipt and continued guarded real-cloud
acceptance. Its complete-account inventory then exposed another live response
distinction: InSpace omits `display_name` from both list and exact reads of
active VM primary disks. Those rows retain a complete OS-disk identity
(`status: Active`, positive user, billing and size fields,
`source_image_type: OS_BASE`, the OS image, storage-pool UUID, timestamps,
`read_only_bootable: false`, and `snapshots`) and correlate one-to-one with a
VM `vda` storage row marked `primary: true`. The dynamically provisioned CSI
disk remains a named `EMPTY` non-primary disk.

RC.12 accepts an omitted disk name in the destructive E2E inventory only for
that complete active OS-base shape. A present name must remain non-empty text,
and an unnamed `EMPTY`, partial, malformed, or duplicate disk row still fails
closed. Complete-account comparison continues to use canonical disk UUIDs, so
boot disks remain visible to baseline and teardown accounting rather than
being silently dropped.

CSI mutable-volume authority is independently tightened. Ordinary reads and
every attach, detach, and delete path require the exact disk UUID and billing
account, a non-empty name, `source_image_type: EMPTY`, no source image, and a
non-bootable disk. Authoritative VM-storage reads reject the target UUID
wherever it is marked primary, including detach recovery after restart.
Create/adoption readback requires the same EMPTY, unsourced, non-bootable
shape; the create response remains diagnostic and its documented omission of
`display_name` does not become ownership authority. Delete tombstone and
absence recovery retain their existing UUID, billing, list-omission, and
durable three-observation proofs after a previously authorized delete.

This does not change supported dynamically provisioned RWO volumes. Manually
constructed or imported CSI volume handles that point at an unnamed, OS-base,
image-backed, read-only-bootable, or VM-primary disk are now rejected; those
handles were never part of the documented dynamic-provisioning contract.
Regression coverage proves the complete live boot-row shape is readable,
incomplete variants fail closed, named EMPTY CSI disks remain discoverable,
and forbidden disk classes cannot issue cloud mutations. Stable promotion
remains gated on a complete guarded RC.12 real-cloud run and exact
account-baseline teardown.
