## Breaking upgrade note

This candidate changes immutable fixed-node cloud-init and advances the
control-plane and bastion ownership schema from v3 to v4. Existing v0.4.0
fixed VMs are not adopted by the v0.4.1 controller. Destroy and recreate the
test cluster when moving to this release; teardown continues to recognize the
older v3 ownership records safely.

Karpenter bootstrap advances from v8 to v9, so existing workers are eligible
for drift replacement through the normal NodeClaim lifecycle.

## Ubuntu package and DNS bootstrap

- Configures TOT as the primary Ubuntu mirror and KKU as the request-failure
  fallback for `noble`, updates, backports, and security packages.
- Uses APT's ordered `mirror+file` transport instead of rewriting whichever
  stock archive URL happens to be present in the image.
- Replaces the DHCP-provided resolver with static Google DNS (`8.8.8.8` and
  `8.8.4.4`) and stops and masks `systemd-resolved`.
- Applies the same policy to the bastion, all three control planes, Karpenter
  workers, and the E2E repair path.

## Release acceptance

- Extends unit and static E2E contracts to verify the exact mirror order,
  shared update/security source configuration, static resolver contents, and
  disabled resolver service.
- Keeps the public NLB acceptance resource short-lived: the E2E test removes
  it immediately after its CCM checks while preserving the remaining cluster
  for later assertions.
